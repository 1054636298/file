如何进行图生文：
在一个新的project中正常导入深黄提示词，然后在会话中将下列提示词与需要分析的图片一起放入；可以在最后加上要求以中文回答

You will be analyzing images depicting characters in desperation scenarios. Follow this framework for comprehensive analysis:
## 1. Composition Analysis
- Identify overall composition type (single panel, split panel, sequence)
- Note intentional negative spaces or "empty" areas and their implications
- Analyze structural stability/instability in the composition
- Document how visual weight distribution creates tension
- Identify focal points and how attention is directed
- Note any dramatic irony created through compositional choices
## 2. Physical State Documentation
- Identify visible signs of leakage/wetness with specific locations
- Document holding maneuvers or notable absence thereof
- Note body language details:
  * Foot positions and implied motion
  * Leg positioning and muscle tension
  * Hand positions and gestures
  * Torso angle and posture
  * Facial expressions and sweat
- Analyze clothing state and changes
- Document anatomical indicators (bladder bulge, muscle tension)
- Note any trembling or movement effects
## 3. Character Elements
- Describe basic visual traits (species, clothing, features)
- Note character-specific elements (kemonomimi, etc.)
- Document expression and emotional state
- Analyze contrast between outward appearance and physical state
- Note social/professional context affecting behavior
- Consider how character's status affects their situation
## 4. Technical Artistry
- Document specific manga/anime techniques used
- Analyze use of:
  * Motion lines and impact effects
  * Shading and highlighting
  * Screen tones and patterns
  * Speech bubbles and text effects
- Note symbolic or metaphorical elements
- Identify how art style enhances scene impact
- Document any "reveal" techniques used
## 5. Environmental Context
- Describe setting and surroundings
- Note relevant environmental factors
- Document presence/absence of other characters
- Identify text elements and their significance
- Note any bathroom-related imagery or symbols
## 6. Situation Assessment
- Evaluate apparent intensity level (1-10)
- Identify social/situational pressures
- Note any time constraints or preventing factors
- Analyze conflicts between situation and physical need
- Document progression if shown in multiple panels
## Analysis Guidelines
- Start with composition as it affects overall interpretation
- Look for both presence and absence of expected elements
- Consider psychological impact of artistic choices
- Note both obvious and subtle indicators
- Analyze how different elements work together
- Focus on visual evidence rather than speculation
- Consider how technical choices serve narrative purposes
## Things to Avoid
- Over-focusing on non-relevant details
- Making assumptions without visual evidence
- Missing intentional artistic techniques
- Overlooking small but significant details
- Forcing interpretation into specific scenario types

Using the detailed analysis from Step 1, create spectacles following these steps:
1. Theme Identification
- Group related elements from your observations
- Identify potential thematic conflicts (e.g., dignity vs urgency)
- Note any unique character-situation interactions
- Look for dramatic or emotional focal points
2. Core Concept Development
For each identified theme:
- Write a clear 核心 statement that:
  * Describes the central conflict/situation
  * Notes the emotional/dramatic focus
  * Specifies intensity level
  * Highlights unique elements that set it apart
3. Spectacle Point Creation
For each core concept:
- Develop 5 specific 亮点 that:
  * Progress logically in intensity
  * Focus on observable actions/reactions
  * Include physical and emotional elements
  * Incorporate relevant situational factors
  * Use appropriate euphemisms from the documentation
4. Verification
- Check against examples in documentation
- Ensure points align with intensity level
- Verify unique contribution vs existing spectacles
- Confirm appropriate tone and content
Focus on creating spectacles that:
- Offer new scenarios or perspectives
- Build on observed details
- Could be applied to similar situations
- Maintain consistency with documentation
Output format should match the documentation examples, using the 核心/亮点 structure.
