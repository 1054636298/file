已得到@大黑塔 授权网页化！
模拟宇宙：绝望校园也是复杂的任务，毕竟随机性很大，而这时候，网页化的优点就体现出来。
本次网页化考虑到任务可以作为虚拟游戏，网页开发两个游戏版本
1.DM助手版（版本代码：V1） 作为现实游戏的辅助工具，充当DM（游戏主持人）
随机事件生成
游戏进程追踪
牌库管理
规则查询
2.虚拟体验版 (版本代码：V2) 完全虚拟的角色和体验游戏
虚拟角色创建
沉浸式游戏体验
服装系统
事件互动
本次用到了vue开发，所以本地开源的压缩文件不仅有源代码（vue文件），也提供了编译后的文件，即dist目录下，可直接在浏览器运行。
同时，本项目已在cloudflare pages部署：https://page2.01101.top
*本项目大多数代码为cursor开发，且开发过程中gemini2.5pro，o3，claude4等模型混用，可能会有一些代码逻辑不完善和错误，请及时反馈！
以下是gemini2.5pro写的readme.md项目介绍
  1. # 模拟宇宙：绝望校园 - 网页版

  3. 本项目是 tabletop game **《模拟宇宙：绝望校园》** 的网页实现版本，基于 Vue.js 2.x 开发。它旨在将原游戏的复杂规则数字化，并提供两种独特的游玩模式，以满足不同玩家的需求。

  5. ## 项目核心特性

  7. - **双核驱动模式**:
  8.   - **V1 - DM助手版**: 一个功能强大的游戏主持人（DM）控制面板。它并非游戏本身，而是为现实生活中的游戏主持人量身打造的辅助工具，能帮助DM实时追踪玩家状态、引导游戏流程、处理复杂规则、并控制所有游戏事件的发生。
  9.   - **V2 - 虚拟体验版**: 一个全自动化的单人游戏模拟器。玩家可以直接在该模式下创建角色，并由程序驱动，完整体验一天充满挑战的校园生活。

  11. - **高精度规则模拟**:
  12.   - 完整实现了游戏核心的**时间轴系统**，精确到每一堂课、每一次课间。
  13.   - 实现了复杂的**服装系统**，包括随机抽取、部位搭配、以及决定玩家能否进入卫生间的“得体着装”判定。
  14.   - 实现了**膀胱值**、**宇宙碎片**、**奇物**、**状态效果**等核心资源与状态系统。

  16. - **智能化的DM引导**:
  17.   - V1 DM面板能够根据当前游戏阶段（上课、课间、午休、甚至最终决战），为DM提供**明确、具体、高亮**的行动指引和规则提示。
  18.   - 实现了**最终决战**的完整分支剧情引导，DM可以选择不同剧本，并按步骤控制剧情发展，完美复现了原规则的精髓。

  20. ## 技术栈

  22. - **核心框架**: Vue.js 2.6.14
  23. - **UI 组件库**: Element UI 2.15.14
  24. - **状态管理**: Vuex 3.5.1
  25. - **前端路由**: Vue Router 3.5.2
  26. - **数据持久化**: `vuex-persistedstate` (利用 LocalStorage)

  28. ## 如何开始

  30. ### 环境要求
  31. - Node.js `12.x` 或更高版本
  32. - Npm `6.x` 或更高版本

  34. ### 安装与运行
  35. 1.  **克隆项目到本地**:
  36.     ```bash
  37.     git clone <your-repo-url>
  38.     ```
  39. 2.  **进入项目目录**:
  40.     ```bash
  41.     cd <project-folder>
  42.     ```
  43. 3.  **安装项目依赖**:
  44.     ```bash
  45.     npm install
  46.     ```
  47. 4.  **启动本地开发服务器**:
  48.     ```bash
  49.     npm run serve
  50.     ```
  51.     启动后，在浏览器访问 `http://localhost:8081` (或其他指定的端口)即可看到应用。

  53. ## 游玩模式详解

  55. ### V1 - DM助手版 (`/v1/dm-panel`)
  56. 此模式是为**游戏主持人（DM）** 设计的。DM可以在此面板上：
  57. - **实时监控**: 查看玩家的各项数值、当前穿着、持有的奇物。
  58. - **控制时间**: 启动、暂停、重置一个与现实时间同步的游戏内倒计时。
  59. - **引导流程**:
  60.     - “游戏进程控制”模块会根据时间轴，清晰地展示当前阶段（如“区域1 - 上课”）、阶段描述以及详细的**规则提示**。
  61.     - DM可以根据提示，点击下方的按钮（如“执行饮水”、“执行如厕”）来为现实中的玩家执行操作。
  62.     - 当游戏进入**最终决战**，面板会自动切换为剧本选择和分步引导模式。
  63. - **使用工具箱**: 手动为玩家抽取事件卡、添加/移除奇物、调整宇宙碎片数量。

  65. ### V2 - 虚拟体验版 (`/v2/character-creation`)
  66. 此模式是一个**单人游戏模拟器**。玩家的体验流程如下：
  67. 1.  **创建角色**: 输入昵称，并**抽取初始服装**。
  68. 2.  **开始游戏**: 进入游戏主界面，所有进程由程序自动推进。
  69. 3.  **做出选择**: 在上课或课间时，点击下方的按钮（如“饮水”、“如厕”、“事件”）来做出选择。
  70. 4.  **即时反馈**: 你的每一个关键操作（特别是“如厕”）都会有成功、失败或警告的弹窗提示。
  71. 5.  **管理装备**: 你可以随时进入“服装”和“奇物”页面来管理你的装备。

  73. ## 核心游戏规则摘要

  75. - **胜利条件**: 顺利度过游戏中的一整天。
  76. - **失败条件**: 膀胱值达到上限，或是在特定情况下无法更换衣物。
  77. - **服装系统**:
  78.     - 衣服分为上衣、下装、打底裤、袜子、鞋子五个部位。
  79.     - 前往卫生间**必须**五个部位都穿着得体的衣物（不能是“皇帝的新衣”）。
  80. - **时间轴**:
  81.     - 游戏严格按照预设的时间轴进行，包括8个常规“上课-课间”区域和多个挑战区域。
  82.     - **上课**: 玩家需要做出“饮水/遭遇”和“事件/冒险”的选择。
  83.     - **课间**: 玩家需要做出“如厕/休息”的选择。
  84. - **如厕机制**:
  85.     - 必须穿着得体。
  86.     - 有50%的几率成功进入厕所，若失败，则下次课间必定成功。
  87.     - 成功进入后，膀胱值会大量减少（但不会清零）。

  89. ## 项目结构

  91. ```
  92. .
  93. ├── public/              # 静态资源
  94. ├── src/
  95. │   ├── components/      # 公共组件
  96. │   ├── config/          # 游戏核心数据 (时间轴, 卡牌数据等)
  97. │   ├── layouts/         # 布局组件
  98. │   ├── plugins/         # 插件配置 (如 Element UI)
  99. │   ├── router/          # 路由配置
  100. │   ├── store/           # Vuex 状态管理
  101. │   │   └── modules/     # (artifacts, clothes, game, user...)
  102. │   ├── styles/          # 全局样式
  103. │   ├── utils/           # 工具函数
  104. │   └── views/           # 页面视图 (V1 和 V2 的主要页面)
  105. ├── README.md            # 项目说明文档
  106. └── package.json         # 依赖配置
  107. ```

  110. - **原游戏规则**: 《模拟宇宙绝望校园》, by 大黑塔
